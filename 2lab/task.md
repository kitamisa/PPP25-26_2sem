# Функциональный API для работы с полигонами

## Цель работы

Разработать API для генерации, преобразования и визуализации плоских полигонов, применяя принципы **функционального программирования**: активное использование функций высшего порядка, модулей `itertools` и `functools`, итераторов и декораторов.

Полигон представлен как кортеж координат вершин, например:

```python
((0, 0), (0, 1), (1, 1), (1, 0))  # квадрат
```

Последовательности полигонов представлены в виде **итераторов**.

***

## Базовое задание (обязательно)

Реализовать ядро API, обеспечивающее генерацию, трансформацию, фильтрацию и визуализацию последовательностей полигонов в функциональном стиле.

### Требования к реализации

- Полигоны представлены как **кортежи кортежей** координат; последовательности полигонов — как **итераторы**.
- Вся обработка реализуется через функции высшего порядка: `map`, `filter`, `functools.reduce`.
- Активно используются функции модуля `itertools`.
- Архитектура **расширяема**: добавление новых генераторов, трансформаций и фильтров не требует изменения существующего кода.

### Обязательные компоненты

#### 1. Визуализация

Функция визуализации последовательности полигонов из итератора (например, с использованием `matplotlib.patches`).

#### 2. Генераторы бесконечных последовательностей

Функции, генерирующие бесконечную последовательность непересекающихся полигонов:

- `gen_rectangle` — прямоугольники;
- `gen_triangle` — треугольники;
- `gen_hexagon` — правильные шестиугольники.

С их помощью с использованием функций `itertools` сгенерировать и визуализировать семь фигур всех трёх типов.

Функции, применимые к последовательности полигонов через `map`:

- `tr_translate` — параллельный перенос;

- `tr_symmetry` — симметрия;

#### 4. Визуализация трансформаций

С помощью функций из пп. 2–3 создать и визуализировать:

- три параллельные «ленты» под острым углом к оси абсцисс;
- две пересекающиеся «ленты», пересекающиеся не в начале координат;
- две параллельные ленты треугольников, симметричных друг другу;
- последовательность четырёхугольников в разном масштабе, ограниченных двумя прямыми через начало координат.

#### 5. Фильтры (минимум 2 из 6)

Функции, применимые через `filter`:

- `flt_convex_polygon` — выпуклые многоугольники;
- `flt_angle_point` — фигуры с углом в заданной точке;
- `flt_square` — фигуры с площадью меньше заданной;
- `flt_short_side` — фигуры с кратчайшей стороной меньше заданного значения;
- `flt_point_inside` — выпуклые многоугольники, содержащие заданную точку;
- `flt_polygon_angles_inside` — выпуклые многоугольники, содержащие хотя бы один угол заданного полигона.

#### 6. Применение фильтров (минимум 1 из 3)

- Фильтрация фигур из п. 4 — на выходе ровно шесть фигур;
- из ≥ 15 фигур разного масштаба отобрать ≤ 4 с кратчайшей стороной меньше заданного значения;
- из ≥ 15 пересекающихся фигур отфильтровать пересекающиеся.

#### 7. Декораторы (минимум 1 из 2 групп)

Декораторы, фильтрующие или преобразующие полигоны в итераторах среди аргументов функции:

- на основе фильтров из п. 5: `@flt_convex_polygon`, `@flt_angle_point`, `@flt_square`, `@flt_short_side`, `@flt_point_inside`, `@flt_polygon_angles_inside`;
- на основе трансформаций из п. 3: `@tr_translate`, `@tr_rotate`, `@tr_symmetry`, `@tr_homothety`.

#### 8. Реализовать функции и продемонстрировать их корректность:

- поиск угла, самого близкого к началу координат (`agr_origin_nearest`);
- поиск самого длинной стороны многоугольника (`agr_max_side`);
- поиск самой маленькой площади многоугольника (`agr_min_area`);
- расчет суммарного периметра (`agr_perimeter`);
- расчет суммарной площади (`agr_area`);

которые можно применить к последовательности полигонов с помощью функции `functools.reduce`.

#### 9. Реализовать функции и продемонстрировать пример их работы (если возможно, с визуализацией):

- склейки полигонов в одну последовательность полигонов из нескольких последовательностей полигонов `zip_polygons(iterator1, iterator2, [iterator3, ...])`. Пример:  
    ```zip_polygons([((1, 1), (2, 2), (3, 1)), ((11, 11), (12, 12), (13, 11))], [((1, -1), (2, -2), (3, -1)), ((11, -11), (12, -12), (13, -11))])``` →
    ```[((1, 1), (2, 2), (3, 1), (1, -1), (2, -2), (3, -1)), ((11, 11), (12, 12), (13, 11), (11, -11), (12, -12), (13, -11))]```.
