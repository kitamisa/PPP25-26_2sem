
## Общие требования ко всем вариантам
- Интерфейс: только консольный ввод и вывод.
- Программа должна корректно обрабатывать ошибочный ввод пользователя.
- Запрещено использовать графический интерфейс и сторонние игровые библиотеки.

## Вариант 1. Камень-ножницы-бумага до 3 побед

### Постановка задачи

Разработать консольную игру «Камень-ножницы-бумага», в которой пользователь играет против компьютера. Компьютер выбирает ход случайным образом, а игра продолжается до тех пор, пока один из участников не наберет 3 победы. Программа должна выводить результат каждого раунда и текущий счет. Простые раундовые терминальные игры часто используются как учебные проекты для отработки условных операторов, циклов и обработки команд.[^1][^2][^3]

### Функциональные требования

- Игрок вводит один из допустимых вариантов хода.
- Компьютер выбирает ход случайно.
- Для каждого раунда определяется победитель.
- Ведется счет побед игрока и компьютера.
- Матч завершается при достижении 3 побед одной из сторон.
- После завершения можно начать новый матч.


### Что проверяется

| Критерий | Баллы |
| :-- | --: |
| Ход компьютера генерируется случайно | 8 |
| Логика сравнения ходов работает верно | 15 |
| Счет матча ведется корректно | 10 |
| Команды и неверный ввод обрабатываются правильно | 10 |
| Код разделен на логические функции | 7 |
| Интерфейс удобен для проверки | 10 |
| **Итого** | **60** |

### Минимум для зачета

- Один раунд отрабатывает правильно.
- Матч идет до 3 побед.
- Итоговый победитель определяется корректно.
- Ошибка ввода не завершает программу аварийно.


### Возможные команды пользователя

- `rock`
- `scissors`
- `paper`
- `score` — показать счет.
- `restart` — начать матч заново.
- `exit` — выйти.

## Вариант 2. Лабиринт в консоли

### Постановка задачи

Разработать консольную игру «Лабиринт». Игрок перемещается по карте, состоящей из символов, от стартовой позиции к выходу. Стены проходить нельзя. Управление осуществляется командами `w`, `a`, `s`, `d`, а после каждого хода программа должна заново выводить текущее состояние карты. Терминальные лабиринты хорошо подходят как учебные проекты, потому что опираются на двумерный массив, проверку границ и обработку команд перемещения.[^5][^1]

### Функциональные требования

- Карта хранится в виде двумерной структуры.
- На карте есть старт, стены, пустые клетки и выход.
- Игрок перемещается по допустимым направлениям.
- При попытке пройти сквозь стену выводится сообщение об ошибке.
- Игра завершается при достижении выхода.
- После завершения можно начать заново.


### Что проверяется

| Критерий | Баллы |
| :-- | --: |
| Карта отображается и обновляется корректно | 12 |
| Перемещение игрока работает без ошибок | 12 |
| Столкновение со стенами и границами обрабатывается верно | 12 |
| Команды управления проверяются корректно | 8 |
| Код структурирован и не перегружен | 8 |
| Программа удобна для демонстрации | 8 |
| **Итого** | **60** |

### Минимум для зачета

- Игрок видит карту и может двигаться по ней.
- Нельзя выйти за пределы карты или пройти через стену.
- Выход из лабиринта завершает игру.
- Программа не падает при неверной команде.


### Возможные команды пользователя

- `w` — вверх.
- `a` — влево.
- `s` — вниз.
- `d` — вправо.
- `restart` — начать заново.
- `exit` — выйти.
